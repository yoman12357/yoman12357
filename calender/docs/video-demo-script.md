# Video Demo Script

## 1-2 Minute Walkthrough

**0:00 - 0:15**  
This project started as a frontend engineering challenge to build an interactive wall calendar component, then evolved into a more polished product-style experience with stronger UX, accessibility, and real-world interaction features.

**0:15 - 0:35**  
Start on desktop and show the wall-calendar layout: the hero image, adaptive accent styling, light and dark theme toggle, and the smooth month transition animations when moving between months.

**0:35 - 0:55**  
Demonstrate date selection by clicking once for a start date, again for an end date, and then show drag-to-select to highlight a range dynamically like a booking flow. Call out the quick actions such as Today, Clear Selection, Select This Week, and previous or next month navigation.

**0:55 - 1:15**  
Show the notes workflow by saving a note for a selected range, then point out that the notes panel stays sticky on desktop so it remains visible while interacting with the calendar. Mention localStorage persistence and the note indicators shown directly on the calendar.

**1:15 - 1:35**  
Create a reminder, highlight the timezone-aware reminder messaging, and explain that the app uses browser notifications when available with an in-app fallback if permission is denied. Mention hover states, toasts, and polished feedback throughout the interaction flow.

**1:35 - 1:55**  
Resize to mobile and show the stacked responsive layout, touch-friendly date cells, usable action buttons, and the same core experience working cleanly on smaller screens. Close by mentioning keyboard navigation, visible focus states, and screen-reader-friendly labeling as part of the accessibility pass.
